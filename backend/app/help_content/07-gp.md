# Área de Gestão de Projeto (GP)

Entrada: `/projects` → `/projects/:id` — Guard: `ProjectMember` aceito + `RequireProjectSetup` em rotas pipeline.

GP é **soberano do projeto** (§4.1 emenda 2026-04-19): tem acesso a **todas** as funcionalidades, inclusive CodeGen e testes — a separação com Dev/Tester/QA é de dia-a-dia, não de permissão.

## 1. Meus Projetos (`/projects`)

- Lista de projetos onde o usuário é membro ativo (status + papel + progresso).
- Indicador de pendências por projeto.
- Navegação direta para o dashboard do projeto.

## 2. Dashboard do Projeto (`/projects/:id`)

- **Scores OCG** em radar chart (P1-P7).
- **Distribuição detalhada** por pilar com adherence_level.
- **Status** (READY/NEEDS_REVIEW/AT_RISK/BLOCKED) + versão do OCG.
- **Stack recomendada** resumida por camada.
- **Billing de IA do projeto**: total_cost_usd, total_tokens, calls + breakdown por operação e provedor.
- **Equipe do projeto** com papel de cada membro.
- Atalhos para todas as abas do pipeline.

## 3. Configurações (`/projects/:id/settings`)

Abas internas:

- **Questionário Técnico** — lista questionário externo submetido + análise + correções via PDF.
- **Repositório Git** — conexão (GitHub/GitLab/Bitbucket), PAT criptografado, branch default.
- **Repos Externos** — atalho para `/external-repos`.
- **Provedor IA do projeto** — chaves próprias (§6.5 compartimentalização):
  - Adicionar/remover provedor.
  - Validar chave em tempo real.
  - Definir padrão do projeto.
- **Gestão de Equipe**:
  - Convidar Dev/Tester/QA/GP com link email.
  - **GP convida outro GP** do mesmo projeto (MVP 11.1).
  - **Transferir soberania GP → GP** (MVP 11.2).
  - Revogar convite pendente, remover membro.
  - Guard de "último GP" na remoção.

## 4. OCG (`/projects/:id/ocg`)

- Visualização completa das 12 seções + versão + `context_health`.
- Histórico de versões (delta log).
- Snapshot de versão antiga (read-only).
- **Rollback formal** — cria nova versão com `trigger_source='rollback'` (MVP 14.7).
- **Consolidação explícita** — recalcula `COMPOSITE_SCORE` (MVP 14.8).
- Health (depth/confidence/quality).
- Regenerar OCG (descarta delta e refaz do questionário — **com confirmação**).
- Tabela `DATA_MODEL` com tabelas + índices + FKs (DT-076).

## 5. Repositórios Externos (`/projects/:id/external-repos`)

- Adicionar repo legado (URL + credenciais).
- Análise automática: stack detectado, vulnerabilidades, compatibilidade GCA.
- 3 abas: Stack Detectado · Segurança · Compatibilidade GCA.
- Roadmap de adaptação passo-a-passo.
- Dispara Arguidor com perguntas específicas sobre o legado.

## 6. Ingestão (`/projects/:id/ingestion`)

- Drop zone (PDF, DOCX, XLSX, PNG, JPG, MD — máx 50 MB).
- Vinculação opcional a módulo do Roadmap (auto-detecta PDF template GCA).
- Pipeline assíncrono Celery com barra de progresso.
- Extração rica por tipo.
- Quarentena PII (libera ou descarta).
- Delete com cascata no OCG.
- Detalhe: análise do Arguidor + gaps + show_stoppers + poor_definitions + improvement_suggestions + module_candidates + ocg_fields_to_update.
- Polling adaptativo (2s processando / 15s idle).

## 7. Gatekeeper (`/projects/:id/gatekeeper`)

- 7 pilares com score + status.
- Items: gaps, show_stoppers, poor_definitions, improvement_suggestions.
- Módulos candidatos (derivados da ingestão).
- Summary + indicador de bloqueadores.
- Health do OCG.

## 8. Arguidor (`/projects/:id/arguider`)

- Lista de perguntas dirigidas geradas a partir da ingestão/legado.
- GP responde (texto + evidência opcional).
- Ignorar com motivo.
- Resolver → alimenta OCG de volta (expand/update).
- Contexto OCG visível (score, status, versão, confiança).

## 9. CodeGen (`/projects/:id/codegen`)

- IDE-like com sidebar Git (árvore de arquivos).
- Sidebar colapsável (toggle).
- Gerar scaffold a partir de `OCG.STACK_RECOMMENDATION`:
  - 9 linguagens canonicamente suportadas (ver [cap. 8](?section=08-codegen)).
  - Injeção automática de DDL do `DATA_MODEL`.
- Preview antes de commit (diff completo).
- Apply → commit no Git do projeto.
- Regenerar arquivo específico.
- Docstrings obrigatórias em todo código gerado.
- Validação pós-geração (pyflakes / esprima / ast.parse / cmake).

## 10. QA Readiness (`/projects/:id/qa`)

- Requisitos de teste derivados de `OCG.TESTING_REQUIREMENTS`.
- Cobertura: unit, integration, e2e, regression, load, security.
- Execução com timeout configurável + logs JSONL por run.
- Gate `quality_gate` conforme Q46.
- QA formal conforme Q47.

## 11. Tester Review (`/projects/:id/tester-review`)

- Abas por tipo: **Specs**, Unit, Integration, E2E, Regression, Load, Security.
- `TestArtifactCard`: título, tipo, status, conteúdo, ações.
- Specs tab:
  - Geração via Ollama (unit/integration/e2e).
  - Geração via Premium (security/compliance).
  - Aprovação/rejeição.
- Modal com visualização + edição + histórico + rejeição.
- Logs de execução toggleable.
- **Stale detection** — banner quando OCG mudou depois da última geração (MVP 10.4).

## 12. Backlog (`/projects/:id/backlog`)

- Backlog vivo derivado do OCG.
- `source_version` rastreia de qual versão do OCG veio cada item.
- Filtros por categoria/status/prioridade.
- Regeneração forçada quando OCG muda.

## 13. Roadmap (`/projects/:id/roadmap`)

- Módulos em 8 categorias canônicas (MVP 9): Foundation, Auth, Data, Business, Infra, UI, Integration, Compliance.
- Progresso geral.
- Detalhamento on-demand via Ollama local.
- Curadoria Premium (readiness + DAG) com WebFetch.
- Plano de deploy com export Markdown.
- Ciclo reativo: GP responde Arguidor → ingestão → módulo vira `adicionado` → deliverable automático.

## 14. Documentação Viva (`/projects/:id/docs`)

- Doc completa do projeto regerada automaticamente.
- Geração por módulo via Ollama.
- Consolidação geral via Premium.
- Regeneração granular por tipo (MVP 10.8).
- Incremento automático em cada commit do pipeline.
- Seção "Modelo de Dados" com DDL inline (DT-076 F5).
- Stale detection com banner.
- Viewer read-only dos documentos originais ingeridos.

## 15. Readiness / Definition of Done (`/projects/:id/readiness`)

- Lista de deliverables por categoria: doc, code, test, process, config, other.
- Status: verified, present, declared, generating, manual_only, missing, waived, error.
- Readiness % por categoria.
- Registry + 7 generators (Fases A-D da Session 21).

## 16. Auditoria do Pipeline (`/projects/:id/audit`)

- Fases executadas no pipeline do projeto (ingestão, gatekeeper, arguider, codegen, qa) com duração + status + context JSON.
- Filtro por fase e janela temporal.

## 17. Backups (`/projects/:id/backups`)

- Snapshots do projeto (OCG + artefatos + DB metadata).
- Criar manual.
- Restore a partir de snapshot.
- Retenção configurável pelo GP.

## 18. Incidentes (`/projects/:id/incidents`)

- Lista de tickets do projeto (compartimentalizado).
- Criar ticket com severidade (BAIXO/MÉDIO/ALTO/CRÍTICO) + descrição + anexos (5 arquivos / 10 MB / 9 extensões).
- Detalhe com comentários + resolução.
- Roteamento automático por papel.
- Contexto obrigatório: `section_reference` (autopreenchido) + `flow_description` (bloqueante).

## 19. Métricas do Projeto (`/projects/:id/metrics`)

- AI usage do projeto (filtrado por `project_id`).
- Eventos de audit com `resource_id=project_id`.
- Autorização: Admin, Support ou membro aceito.

## 20. Ajuda (`/projects/:id/help`)

Este help. Mesmo conteúdo do `/admin/help` — é global da instância, não segmentado por projeto.
