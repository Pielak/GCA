# Área de Gestão de Projeto (GP)

Entrada: `/projects` → `/projects/:id` — acesso para membros ativos do projeto.

O GP é **soberano do projeto**: tem acesso a todas as funcionalidades (OCG, ingestão, Gatekeeper, Arguidor, CodeGen, testes, doc, releases). Dev, Tester e QA compartilham o projeto com papéis de dia-a-dia, mas a permissão plena está com o GP.

## Meus Projetos (`/projects`)

- **Lista** de projetos onde você é membro ativo, com status, seu papel e progresso.
- Indicador de pendências por projeto (items do Arguidor esperando resposta, quarentenas pendentes, etc).
- Clique em um projeto para ir ao dashboard dele.

## Dashboard do Projeto (`/projects/:id`)

Visão consolidada:

- **Scores OCG** em radar chart dos 7 pilares (P1–P7).
- **Detalhamento** por pilar com nível de aderência.
- **Status** do projeto: READY, NEEDS_REVIEW, AT_RISK ou BLOCKED + versão atual do OCG.
- **Stack recomendada** resumida por camada.
- **Billing de IA do projeto**: custo total em USD, tokens, número de chamadas + breakdown por operação e provedor.
- **Equipe** com papel de cada membro.
- Atalhos para todas as abas do pipeline.

## Configurações (`/projects/:id/settings`)

Abas internas:

### Questionário Técnico
- Questionário submetido inicialmente + análise + correções enviadas pelo GP depois do Arguidor.
- PDF alternativo para preenchimento offline.

### Repositório Git
- Conecta GitHub, GitLab ou Bitbucket.
- Token de acesso pessoal (PAT) criptografado.
- Define branch default.
- **Um mesmo repositório não pode ser compartilhado entre projetos** — o sistema bloqueia e mostra qual projeto já usa aquele repo.

### Provedor de IA do projeto
- **Adicionar/remover** provedor: Anthropic, OpenAI, DeepSeek, Ollama.
- **Validar a chave** antes de salvar.
- **Definir o padrão** do projeto.
- Essas chaves são **separadas** das globais da instância — custo da IA do dia-a-dia fica com o projeto.

### Equipe
- Convidar Dev, Tester, QA ou outro GP por email.
- **Convidar outro GP** do mesmo projeto (co-gestão).
- **Transferir a soberania** para outro membro ativo.
- Revogar convite pendente.
- Remover membro (com proteção de último GP).

### Repos Externos
- Atalho para `/projects/:id/external-repos` (ver abaixo).

## OCG (`/projects/:id/ocg`)

Página central do projeto. Mostra as 12 seções do OCG renderizadas, a versão atual e o `context_health`.

O que você pode fazer aqui:

- **Ver o OCG atual** com detalhamento por seção.
- **Linha do tempo de versões** — histórico de mudanças com origem de cada delta.
- **Ver snapshot** de qualquer versão anterior (somente leitura).
- **Rollback** — voltar para uma versão anterior (cria nova versão, não apaga histórico).
- **Consolidar OCG** — recalcular score composto e status a partir dos pilares. Idempotente.
- **Regenerar OCG** — descartar histórico e refazer do zero (ação destrutiva, com confirmação dupla).
- **Modelo de dados** — tabela com tabelas, índices e FKs derivados do OCG.

Detalhes em [cap. 5 — OCG](?section=05-ocg).

## Repositórios Externos (`/projects/:id/external-repos`)

Vincula repositórios legados ao projeto para o GCA analisar:

- **Adicionar** repo com URL + credenciais.
- O sistema analisa automaticamente: stack detectado, vulnerabilidades, compatibilidade com o GCA.
- **3 abas** no resultado:
  - **Stack Detectado**: linguagem, frameworks, arquivos totais, Docker, CI/CD, testes.
  - **Segurança**: nível de risco (baixo/médio/alto) + lista de vulnerabilidades com versão recomendada.
  - **Compatibilidade**: status geral (compatível / requer adaptação / incompatível), esforço estimado, breakdown por área.
- **Roadmap de adaptação** passo-a-passo.
- A análise do legado alimenta o Arguidor com perguntas específicas.

## Ingestão (`/projects/:id/ingestion`)

- **Drop zone** para upload (PDF, DOCX, XLSX, PNG, JPG, MD — até 50 MB).
- **Vinculação opcional** a um módulo do Roadmap (útil para anexar documentação específica).
- Pipeline assíncrono com **barra de progresso** em tempo real: `queued → extracting_text → analyzing → updating_ocg → regenerating_backlog → completed`.
- **Extração rica** por tipo (DOCX com tabelas, PDF com AcroForm + OCR, XLSX preservando estrutura).
- **Quarentena PII** — documentos com CPF/CNPJ/cartão/telefone ficam retidos até o GP decidir: liberar ou descartar.
- **Detalhe do documento** — análise do Arguidor + gaps + show_stoppers + poor_definitions + improvement_suggestions + módulos candidatos.
- **Deletar** documento com cascata no OCG.

## Gatekeeper (`/projects/:id/gatekeeper`)

- **7 pilares** com score + status (ok, warning, blocker).
- **Items rastreados**: gaps, show_stoppers, poor_definitions, improvement_suggestions.
- **Módulos candidatos** derivados das ingestões.
- **Summary** + indicador de bloqueadores ativos.
- **Health do OCG** (confidence, quality).

## Arguidor (`/projects/:id/arguider`)

- **Lista de perguntas dirigidas** geradas a partir das ingestões e da análise de repos legados.
- **Responder** com texto + evidência opcional.
- **Ignorar com motivo** se a pergunta não se aplica ao seu projeto.
- **Resolver** alimenta o OCG de volta (expand ou update).
- **Contexto OCG visível** (score, status, versão, confidence) para você decidir com contexto.

## CodeGen (`/projects/:id/codegen`)

Interface tipo IDE com árvore de arquivos do repositório à esquerda (sidebar colapsável).

- **Gerar scaffold** a partir do `STACK_RECOMMENDATION` do OCG.
- **9 linguagens suportadas** canonicamente + Python via LLM.
- **DDL automático** — schema.sql, seed.sql e migrations injetados a partir do modelo de dados.
- **Preview antes do commit** — diff completo.
- **Apply** — cria commit no repositório Git do projeto com mensagem padrão.
- **Regenerar arquivo** específico.
- **Docstrings obrigatórias** em todo código gerado.
- **Validação pós-geração**: pyflakes, esprima, ast.parse, cmake conforme a linguagem.

Detalhes em [cap. 8 — Codegen](?section=08-codegen).

## QA Readiness (`/projects/:id/qa`)

- Requisitos de teste derivados do `TESTING_REQUIREMENTS` do OCG.
- Cobertura: unit, integration, e2e, regression, load, security.
- **Execução** com timeout configurável; logs ficam em JSONL por run.
- Gate de qualidade conforme definido no questionário.
- QA formal (opcional) conforme configurado.

## Tester Review (`/projects/:id/tester-review`)

- **Abas por tipo**: Specs (planos), Unit, Integration, E2E, Regression, Load, Security.
- **Card de teste** com título, tipo, status e ações (editar, aprovar, rejeitar, executar).
- Specs tab:
  - Geração automática via IA local para unit/integration/e2e.
  - Geração via IA premium para security/compliance.
  - **Aprovação/rejeição** do plano.
- **Modal de detalhe** com visualização, edição, histórico e rejeição.
- **Logs de execução** expansíveis por teste.
- **Banner "Stale"** aparece quando o OCG mudou depois da última geração.

## Backlog (`/projects/:id/backlog`)

- **Backlog vivo** derivado automaticamente do OCG.
- Cada item tem `source_version` — rastreabilidade de qual versão do OCG gerou o item.
- Filtros por categoria, status e prioridade.
- **Regeneração forçada** quando você quer atualizar o backlog manualmente.

## Roadmap (`/projects/:id/roadmap`)

- **8 categorias canônicas**: Foundation, Auth, Data, Business, Infra, UI, Integration, Compliance.
- **Progresso geral** do projeto.
- **Detalhamento sob demanda** de cada módulo (gerado via IA local).
- **Curadoria** de módulos críticos (via IA premium).
- **Plano de deploy** com export Markdown.
- Quando você responde o Arguidor, módulos relacionados ficam marcados como "adicionado" e o deliverable correspondente é criado automaticamente.

## Documentação Viva (`/projects/:id/docs`)

- Documentação do projeto gerada automaticamente.
- Por módulo: via IA local.
- Consolidação geral: via IA premium.
- **Regenerar** granular por tipo.
- Atualização automática a cada commit do pipeline.
- Seção "Modelo de Dados" embute o DDL.
- **Banner "Stale"** avisa quando o OCG mudou e a doc precisa regenerar.
- **Viewer somente-leitura** dos documentos originais ingeridos.

## Readiness / Definition of Done (`/projects/:id/readiness`)

- **Lista de deliverables** por categoria (doc, code, test, process, config, other).
- **Status** por deliverable: verified, present, declared, generating, manual_only, missing, waived, error.
- **% de readiness** por categoria.
- Geração automática de alguns deliverables.

## Auditoria do Pipeline (`/projects/:id/audit`)

- Lista das fases do pipeline que rodaram no projeto (ingestão, Gatekeeper, Arguidor, CodeGen, QA) com duração, status e contexto.
- Filtro por fase e janela temporal.
- Drill-down por evento.

## Backups do Projeto (`/projects/:id/backups`)

- Snapshots do projeto (OCG + artefatos + metadados do banco).
- **Criar backup manual**.
- **Restore** a partir de snapshot selecionado.
- **Retenção** configurável pelo GP.

## Incidentes (`/projects/:id/incidents`)

- Lista de tickets do projeto.
- **Criar ticket** com severidade (BAIXO, MÉDIO, ALTO, CRÍTICO) + descrição + anexos (até 5 arquivos, 10 MB cada, 9 extensões aceitas).
- Detalhe com comentários + resolução.
- **Roteamento automático por papel** — bugs técnicos para Dev, mudança de escopo para GP, sustentação para Support.
- Contexto obrigatório: `section_reference` (autopreenchido conforme a aba de origem) e `flow_description` (bloqueante — você precisa descrever o fluxo que disparou o problema).

## Métricas do Projeto (`/projects/:id/metrics`)

- Uso de IA do projeto (filtrado pelo `project_id`).
- Eventos de auditoria do projeto.
- Autorização: Admin, Support ou membro aceito do projeto.

## Ajuda (`/projects/:id/help`)

Esta página. O conteúdo é o mesmo de `/admin/help` — é documentação global da instância, não varia por projeto.
